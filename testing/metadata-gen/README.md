# Regenerating the zombienet-sdk runtime modules

`testing/zombienet-sdk-tests/tests/codegen/*.rs` are the typed `subxt` clients for the six runtimes
the bridge tests talk to (Rococo, Westend, and their Asset Hub / Bridge Hub system parachains). They
are generated by this repo's `runtime-codegen` tool (in `--full` mode, i.e. with the
`tx()`/`storage()`/`apis()` accessors, equivalent to `#[subxt::subxt(..)]`). subxt validates the
generated calls against each node's live runtime metadata, so these modules **must match the runtime
version embedded in the `polkadot` / `polkadot-parachain` binaries you run** — which is the
polkadot-sdk revision this repo pins in `Cargo.lock`.

This repo builds no runtimes of its own, so the modules are generated from a polkadot-sdk checkout.
`generate.sh` makes that self-contained: it reads the pinned commit from `Cargo.lock`, checks out
polkadot-sdk at exactly that commit, builds the six `*-runtime` WASM blobs, builds `runtime-codegen`,
and writes one `tests/codegen/<chain>.rs` module per runtime.

## Usage

From the repo root:

```bash
# clone polkadot-sdk @ pinned-rev into testing/metadata-gen/polkadot-sdk and generate; the clone is
# kept for reuse (next run skips re-cloning):
testing/metadata-gen/generate.sh

# reuse an existing polkadot-sdk checkout (faster if its runtime WASM is already built):
testing/metadata-gen/generate.sh --polkadot-sdk /path/to/polkadot-sdk

# also remove the cloned checkout when done (default keeps it for reuse):
testing/metadata-gen/generate.sh --cleanup
```

The first run is **slow** — it builds six runtime WASM blobs from a fresh checkout. `--polkadot-sdk`
pointing at a checkout that already has the runtimes built is much faster.

## How it works

- `runtime-codegen` (`tools/runtime-codegen`) loads each runtime WASM via `subwasm`'s `WasmTestBed`,
  extracts its metadata, and runs `subxt-codegen` over it. The `--full` flag emits a complete subxt
  client (accessors + subxt's default derives) — what the tests consume — instead of the
  relay-clients' `runtime_types`-only output.
- Generating straight from the runtime WASM means no extra extraction crate has to be injected into
  the polkadot-sdk checkout, and the metadata is exactly what the pinned runtimes expose.

## When to regenerate

Whenever this repo's pinned polkadot-sdk revision changes (a different commit hash for
`git+https://github.com/paritytech/polkadot-sdk` in `Cargo.lock`), re-run `generate.sh` and commit the
updated `tests/codegen/*.rs` modules.
